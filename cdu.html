Módulo de Registros: Cambios durante el embarazo

Autora: Martínez Castro María José

Nombre: Registrar cambios
Identificador: CU 12
Descripción: El actor principal, Persona Embarazada, interactúa con el sistema para el registro del seguimiento de cambios durante su embarazo.
Precondiciones: Contar con una cuenta del tipo “Persona embarazada”. Haber entrado a la aplicación
Flujo principal:

  1. La persona embarazada inicia sesión en el sistema.
  2. Selecciona en el menú principal (UI 1) la opción de “Registros”.
  3. Selecciona en el menú de registros (UI 2) la opción de “Cambios Físicos”.
  4. Se presenta un formulario (UI 3) para ingresar los datos de los cambios:
    1. Antojos
    2. Cambios en la piel
    3. Frecuencia al orinar
    4. Tipos de cambios en la piel
    5. Diámetro del vientre
    6. Notas
  5. La persona embarazada completa el formulario y lo envía al hacer clic en “Guardar”.
  6. El sistema valida los datos ingresados.
  7. Si los datos son válidos, se almacenan en la base de datos.
  8. Se muestra un mensaje de confirmación al usuario.
Flujos Alternativos:
  • Datos inválidos: Si los datos ingresados no cumplen con los criterios de validación, se muestra un mensaje de error al usuario y se le solicita que corrija la información (UI 4).
  • Cancelación del proceso: La persona embarazada puede salir del proceso en cualquier momento antes de enviar los datos y no se guardará lo que se llenó en el formulario.
Post condiciones:
  • Se ha creado un nuevo registro en la tabla de cambios con los datos ingresados por el usuario.
  • Se ha actualizado el registro del usuario con la fecha del último registro de cambios.

Módulo de Citas

Autora: Mercado Vargas Melisa

Nombre: Agendar cita
Identificador: CU 19
Descripción: Agendar una cita con un médico o una persona embarazada.
Precondiciones: El usuario se ha identificado previamente dentro del sistema.
Postcondiciones: Se habrá agendado una cita.
Flujo básico:

  1. El caso de uso inicia cuando el usuario desea agendar una cita con otro usuario en específico.
  2. Si el usuario es del tipo persona embarazada, se va al flujo alterno A, de lo contrario, si el usuario es del tipo médico, el sistema despliega las fechas, los horarios y las ubicaciones disponibles (UI 20).
  3. El médico selecciona la fecha, hora y ubicación de la cita.
  4. El médico decide si guarda o no la cita. [Flujo alterno B]
  5. El sistema guarda la cita en la agenda del médico y la agrega a la agenda de la persona embarazada.
  6. Termina el caso de uso cuando la cita haya sido agendada.
Flujo alterno A. El usuario es una persona embarazada:
  1. El sistema despliega las fechas y los horarios disponibles para agendar una cita con el médico seleccionado (UI 20).
  2. La persona embarazada selecciona la fecha, hora y ubicación de la cita.
  3. La persona embarazada decide si guarda o no la cita. [Flujo alterno B]
  4. El sistema guarda la cita en la agenda del médico y la agrega a la agenda de la persona embarazada.
  5. Termina el caso de uso cuando la cita haya sido agendada.
Flujo alterno B. El usuario decide no agendar la cita:
  1. El sistema regresa al usuario a la pantalla principal.
  2. Termina el caso de uso.


UI20 Formulario de agenda de cita.

Identificador: CU 22
Descripción: Cancelar una cita previamente agendada con un médico o una persona embarazada.
Precondiciones: El usuario se ha identificado previamente dentro del sistema.
Postcondiciones: Se habrá cancelado la cita previamente agendada.
Flujo básico:

  1. El caso de uso inicia cuando el usuario desea modificar una cita previamente agendada con otro usuario en especifico.

  2. Termina el caso de uso cuando se haya cerrado la ventana emergente.

Descripción: Permite eliminar publicaciones que haya realizado previamente, según sea necesario.

  2. El sistema mostrara el mensaje de “Cancelación exitosa”.
  3. Termina el caso de uso.
